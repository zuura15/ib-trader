# Variant Comparison

## Variant A: Classic Trading Workstation

- Dominant pane: no single pane dominates; monitoring is distributed
- Layout philosophy: many panes visible simultaneously with dense tabsets
- Target operator style: active monitor managing multiple data streams at once
- Strengths: maximum situational awareness, low navigation overhead
- Weaknesses: visually dense, lower hierarchy
- Difference: most traditional and information-heavy of the four

## Variant B: Modern Control Center

- Dominant pane: orders and detail context
- Layout philosophy: cleaner central workflow with contextual side support
- Target operator style: operator who wants hierarchy and controlled attention
- Strengths: cleaner structure, strong visual grouping, easier scanning
- Weaknesses: some panels are less immediate
- Difference: contextual and editorial compared with the denser layouts

## Variant C: Command-Centric Operator View

- Dominant pane: command console
- Layout philosophy: console-first with supporting diagnostics around it
- Target operator style: keyboard-first operator driving workflows by command entry
- Strengths: fast command workflow, tight feedback loop
- Weaknesses: lower passive visibility for non-console data
- Difference: the only variant where command entry visually dominates the workspace

## Variant D: Bot / Automation Supervision

- Dominant pane: bots supervision grid
- Layout philosophy: automation health front and center, with alerts and logs acting as escalation channels
- Target operator style: supervisor of automated strategies and services
- Strengths: strong automation oversight, quick fault detection
- Weaknesses: weaker manual trade management emphasis
- Difference: the only layout with intentionally oversized bot monitoring surfaces
